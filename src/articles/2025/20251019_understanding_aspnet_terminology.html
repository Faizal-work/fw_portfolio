Understanding ASP.NET Terminology 19-October-2025 by Faizal.Work

Some keywords for you to study to stand out (hint: Ask ChatGPT to explain them to you - using Chat in your own time to help with prep is OK, but you need to ask to EXPLAIN so you can discuss them with us):

Term Description
IOptions The Options pattern for strongly-typed configuration access. Binds configuration from appsettings.json to C# classes and injects them into your services. Available in three variants: IOptions (singleton), IOptionsSnapshot (scoped), and IOptionsMonitor (real-time reload).
Middleware Components that form the request/response pipeline in ASP.NET Core. Each middleware can process HTTP requests before and after the next component. Order matters - they execute sequentially (e.g., Authentication before Authorization).
NuGet The package manager for .NET (similar to npm for Node.js or pip for Python). Manages external libraries and dependencies for your projects. Packages are referenced in your .csproj file.
Dependency Injection The built-in IoC (Inversion of Control) container in .NET. Instead of creating dependencies, they are injected into classes. Three lifetimes: Transient (new instance every time), Scoped (one per request), and Singleton (one for app lifetime).
XUnit A modern, popular testing framework for .NET. Uses [Fact] attribute for simple tests and [Theory] for parameterized tests. Follows the Arrange-Act-Assert pattern and includes various assertion methods.
Parameterised Unit Test Data-driven tests that run the same test logic with different input values. In XUnit, implemented using [Theory] attribute with [InlineData] or [MemberData] to provide test cases. Reduces code duplication and tests multiple scenarios efficiently.
NSubstitute A friendly mocking library for .NET testing (alternative to Moq). Creates fake implementations of interfaces to isolate code under test. Uses intuitive syntax like Substitute.For and Received() to verify method calls.
